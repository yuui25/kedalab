# 敵対的サンプル攻撃（Adversarial Examples）

> **このファイルの位置づけ：** ニューラルネットワークへの推論時攻撃の原理。人間には知覚できない微小な入力変化でモデルを意図的に誤分類させる手法群の共通基盤。勾配ベース攻撃とスパース攻撃の区別、ホワイトボックス／ブラックボックスの脅威モデルを把握していないと、各攻撃の条件判断ができない。
> 前提知識 → `Deep_Learning/Neural_Networks.md`（勾配・誤差逆伝播の概念）

---

## 敵対的サンプルとは

### 着火条件

- ニューラルネットワークを使った分類器・検知器に対して、入力を操作して意図的な誤分類を引き起こしたいとき
- モデルへのアクセスレベルがホワイトボックス（勾配取得可能）またはブラックボックス（入出力のみ）のどちらであっても適用できる

### 観点・着眼点

通常の入力 `x` に対して、人間には識別できない微小な摂動 `δ` を加えた `x' = x + δ` を作ると、モデルの出力が大きく変わる現象。

```
人間の目：        猫          猫（ほぼ同じに見える）
モデルの出力：    猫（99%）  → ダチョウ（99.9%）
```

**なぜ起きるか：**
ニューラルネットワークは高次元の統計的パターンで分類する。人間には無意味な微小変化でも、モデルが依拠するシグナルを「損失が最大化する方向」に動かすと、決定境界を越えて誤分類が成立する。

**攻撃が成立する根本条件：**
- 損失関数が入力に対して微分可能（勾配が計算できる）
- 決定境界が入力空間で複雑に折れ曲がっており、わずかな移動で越えられる
- モデルが人間の知覚とは異なる特徴に依存している

## ノルム制約：摂動の「見えなさ」の定量化

### 観点・着眼点

摂動 `δ` の大きさをノルムで制約することで「知覚できない変化」を保証する。使うノルムによって攻撃の性質が変わる。

| ノルム | 定義 | 意味 | 対応する攻撃手法 |
|--------|------|------|----------------|
| **L∞** | `max(|δᵢ|)` | 全要素の最大変化量を ε 以下に抑える | FGSM, PGD |
| **L2** | `√Σ(δᵢ²)` | 全体的な変化量のエネルギーが ε 以下 | C&W Attack |
| **L0** | 変化させた要素の個数 | 変化させる要素の数を最小化する | Sparsity Attacks |

## ホワイトボックス vs ブラックボックス

### 観点・着眼点

モデルへのアクセスレベルで実行可能な攻撃が決まる。評価前にどのレベルにあるかを確認する。

| アクセスレベル | 得られる情報 | 実行可能な手法 |
|--------------|------------|--------------|
| **ホワイトボックス** | モデルの重み・アーキテクチャ・勾配すべて | FGSM, PGD, C&W |
| **グレーボックス** | モデルの種類・アーキテクチャは既知、重みは不明 | 転移攻撃（代替モデルで生成して転移させる） |
| **ブラックボックス** | 入力に対する出力（スコアまたはラベル）のみ | クエリベース攻撃・転移攻撃 |

---

## FGSM（Fast Gradient Sign Method）

### 着火条件

- ホワイトボックス環境でモデルの勾配にアクセスできる場合
- 計算コストを抑えて大量の敵対的サンプルを生成したいとき、または攻撃の基準線として最も単純な手法を試したいとき

### 手順

```
x' = x + ε × sign(∇ₓ J(x, y))
```

1. 入力 `x` と正解ラベル `y` に対する損失 `J(x, y)` を計算する
2. 損失を入力 `x` で微分した勾配 `∇ₓ J` を求める
3. 勾配の符号（`sign`）だけを取り、ε 倍して元の入力に加算する
4. 結果の `x'` を分類器に入力し、誤分類が成立するか確認する

**直観：**「正解クラスに対する損失が最も大きくなる方向」に ε だけ動かす。符号だけ取ることで L∞ 制約 ε 以下に自動的に収まる。

### 注意点・落とし穴

- 1ステップのみの移動のため、強固なモデルや防御済みモデルに対しては成功率が低い
- ε が大きすぎると人間に知覚されるが、小さすぎると誤分類が成立しない。タスクとモダリティに応じた調整が必要
- Adversarial Training 済みモデルにはほぼ通用しない → PGD に切り替える

---

## PGD（Projected Gradient Descent）

### 着火条件

- ホワイトボックス環境で FGSM より強力な敵対的サンプルが必要なとき
- 防御済みモデル（Adversarial Training 等）に対してより高い成功率を求める場合
- モデルの堅牢性を正確に評価したいとき（PGD は堅牢性の標準的な評価指標として使われる）

### 手順

```
x⁽⁰⁾ = x（またはランダムな初期点）
x⁽ᵗ⁺¹⁾ = Proj_ε[ x⁽ᵗ⁾ + α × sign(∇ₓ J(x⁽ᵗ⁾, y)) ]
```

1. 開始点を設定する（元の入力 `x` または ε-ball 内のランダムな点）
2. 1ステップあたり `α` だけ勾配の符号方向に動かす
3. 毎ステップ L∞ ε-ball（`|x' - x| ≤ ε`）に射影して制約内に収める
4. 指定したステップ数（典型：20〜40回）繰り返す
5. 複数の開始点から試行し、誤分類が成立したものを採用する

**FGSM との比較：**

| | FGSM | PGD |
|-|------|-----|
| ステップ数 | 1 | 複数（典型：20〜40） |
| 攻撃強度 | 弱め | 強め |
| 計算コスト | 低 | 高 |

### 注意点・落とし穴

- 反復回数が多すぎると計算コストが増大する。用途に応じてステップ数を調整する
- PGD で学習された Adversarial Training でも、C&W のような最適化ベースの攻撃には弱いことがある

## 転移可能性（Transferability）を使ったブラックボックス攻撃

### 着火条件

- 標的モデルの内部情報（重み・勾配）にアクセスできないブラックボックス環境
- 標的モデルへのクエリ数が制限されているか、入出力ペアのみ取得可能な場合

### 手順

1. 標的 API に入力を送り、入力・出力のペアを収集する
2. 収集したペアを学習データとして代替モデル（Surrogate Model）を訓練する
3. 代替モデルに対してホワイトボックス攻撃（FGSM/PGD）で敵対的サンプルを生成する
4. 生成した敵対的サンプルを標的 API に送信し、転移による誤分類が成立するか確認する

**転移が起きやすい条件：**
- 代替モデルと標的モデルが同じタスク・類似した特徴空間で学習している
- PGD や Ensemble Attack（複数の代替モデルを使う）は FGSM より転移率が高い

### 注意点・落とし穴

- 代替モデルの品質が低いと転移率が大幅に下がる
- クエリ数に制限がある場合、代替モデルの訓練に使えるペアが少なくなり精度が落ちる
- 転移攻撃の成功は保証されない。代替モデルと標的モデルの類似性に依存する

## Sparsity Attacks（L0 制約：疎な攻撃）

### 着火条件

- 変化させる要素数を最小化したい場合（物理的な攻撃・特定フィールドの改ざんに相当する状況）
- ネットワークトラフィックの少数フィールドだけを変更して異常検知を回避したいとき
- 画像の数ピクセルにステッカーを貼ることで誤認を引き起こしたいとき

### 手順（JSMA: Jacobian-based Saliency Map Attack）

1. 入力の各要素に対する損失の感度（ヤコビアン：`∂J/∂xᵢ`）を計算する
2. 感度が最も高い要素（最も影響力が大きいピクセル・フィールド）を特定する
3. その要素のみを変化させ、誤分類が成立するか確認する
4. 成立しない場合は感度上位から順に要素を追加して繰り返す

**L∞ 攻撃との対比：**

| | L∞ 攻撃（FGSM等） | L0 攻撃（Sparsity） |
|-|-----------------|-------------------|
| 変更範囲 | 全要素に薄くノイズ | 少数の要素に大きく変更 |
| 物理的な対応 | 画像全体の微小ノイズ | ステッカー・パッチ |
| 検知のリスク | 統計的に検知困難 | 局所的な異常として検知される可能性がある |

### 注意点・落とし穴

- L0 攻撃は変化を局所に集中させるため、局所的な異常として統計的に検知されやすい
- 1ピクセル（1要素）の変化で成立する Pixel Attack は高次元の分類器では成功率が落ちやすい

## 防御手法と「通用しなくなる条件」

攻撃者として防御手法を把握しておかないと、防御済みモデルで攻撃が通用しない理由を特定できない。

| 防御手法 | 概念 | 通用しなくなる条件 |
|---------|------|----------------|
| **Adversarial Training** | 敵対的サンプルを訓練データに含めて学習する | 訓練に使った攻撃と同じ手法では成功率が落ちる。より強力な攻撃（C&W等）に切り替える |
| **入力平滑化** | 推論前に入力を平滑化してノイズを除去する | 平滑化後も残存するパターンを持つ攻撃を使う |
| **Detection-based Defense** | 敵対的サンプルの統計的特徴を検知して拒否する | 検知器の判定基準を把握した Adaptive Attack で回避する |

---

## 注意点・落とし穴（共通）

- ε の選び方はモダリティに依存する。画像・音声・テキスト・ネットワークトラフィックでそれぞれ「知覚できない変化」の定義が異なる
- デジタル空間でのみ有効な攻撃が、印刷・カメラ撮影を経ると減衰して物理世界では機能しないことがある（Expectation over Transformation で対応）
- テキスト・ネットワークパケットは離散値のため、連続空間で設計した摂動をそのまま適用できない
