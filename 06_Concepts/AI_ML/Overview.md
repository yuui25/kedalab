# 機械学習 概要

> **このファイルの位置づけ：** AI Red Teaming の全攻撃技術（プロンプトインジェクション・敵対的サンプル攻撃・データポイズニング等）を理解するための共通基盤。どの攻撃手法を学ぶ際にも、対象システムがどの学習パラダイムで動いているかを把握することが前提になる。

## 機械学習の種類と選択基準

### 観点・着眼点

**3つの主要パラダイム：**

- **教師あり学習（Supervised Learning）**：ラベル付きデータ（入力と正解のペア）がある。分類・回帰問題に対応。
- **教師なし学習（Unsupervised Learning）**：ラベルなしデータ。データの構造・パターン・クラスタを発見する。
- **強化学習（Reinforcement Learning）**：エージェントが環境と対話しながら報酬を最大化するよう学習する。

**問題の種類による分類：**

| 問題タイプ | 説明 | 例 |
|-----------|------|-----|
| 分類（Classification） | カテゴリを予測 | スパム判定、画像認識 |
| 回帰（Regression） | 連続値を予測 | 価格予測、気温予測 |
| クラスタリング | データをグループ化 | 顧客セグメント |
| 次元削減 | 特徴量を圧縮 | 可視化、ノイズ除去 |
| 異常検知 | 外れ値の検出 | 不正検知、障害検知 |
| 生成モデル | 新しいデータを生成 | 画像生成、テキスト生成 |

### 手順（問題解決の一般フロー）

1. **問題定義**：予測したいものは何か、ラベルがあるか確認
2. **データ収集・確認**：サイズ、特徴量の型、欠損値、クラスバランスを確認
3. **前処理**：正規化/標準化、欠損値処理、カテゴリ変数のエンコーディング
4. **モデル選択**：問題タイプ・データサイズ・解釈性の要件に基づいて選択
5. **訓練**：学習データでモデルをフィット
6. **評価**：テストデータで汎化性能を測定
7. **改善**：ハイパーパラメータチューニング、特徴量エンジニアリング

### 評価指標

**分類問題：**
- **Accuracy（正解率）** = (TP+TN) / 全サンプル数。クラスが不均衡な場合は不適。
- **Precision（適合率）** = TP / (TP+FP)。陽性予測の正確さ。偽陽性を減らしたいとき重視。
- **Recall（再現率）** = TP / (TP+FN)。実際の陽性を見逃さない能力。偽陰性を減らしたいとき重視。
- **F1スコア** = 2 × (Precision × Recall) / (Precision + Recall)。PrecisionとRecallのバランス指標。
- **混同行列（Confusion Matrix）**：TP/TN/FP/FNの全体像を把握。

**補足指標（クラス不均衡・高精度評価が必要な場合）：**
- **Specificity（特異度）** = TN / (TN+FP)。陰性を正しく陰性と判定できる能力。Recall が陽性の検出力ならば、Specificity は陰性の検出力。
- **AUC（Area Under the ROC Curve）**：分類閾値を変化させたときの Recall vs (1-Specificity) の曲線面積。1.0 に近いほど識別能力が高い。閾値に依存しない汎用的な評価が可能。
- **Matthews Correlation Coefficient（MCC）**：TP/TN/FP/FN をすべて考慮した単一スコア。-1〜+1 の範囲で、クラスが極端に不均衡でも信頼性が高い。Accuracy が高くても MCC が低ければ「多数クラスを予測するだけのモデル」と判断できる。

**セキュリティ文脈での指標の選び方：**

| 状況 | 重視する指標 | 理由 |
|------|-----------|------|
| 脅威検知・マルウェア検知 | **Recall（再現率）** | 見落とし（FN = 攻撃を正常と判断）のコストが高い |
| アラートトリアージ・リソース制約がある環境 | **Precision（適合率）** | 誤検知（FP = 正常を攻撃と判断）のフォローアップコストが高い |
| クラス不均衡なセキュリティデータセット | **F1・AUC・MCC** | Accuracy だけでは多数クラス（正常通信）を予測するだけのモデルを見抜けない |

> **落とし穴：** Accuracy: 0.99 でもスパム検知に失敗するケースがある。これは正常メールが 99% を占めるデータで「すべて正常」と予測するだけで高 Accuracy になるため。セキュリティ用途では Accuracy だけを見てモデルを信頼しない。

### classification_report の読み方（多クラス分類）

`sklearn.metrics.classification_report` が出力する各列の意味：

| 列 | 意味 |
|---|---|
| `precision` | そのクラスと予測した中で正解の割合（FP を減らしたいとき重視） |
| `recall` | そのクラスの実サンプルを正しく検出できた割合（FN を減らしたいとき重視） |
| `f1-score` | precision と recall の調和平均 |
| `support` | テストデータ中のそのクラスの実サンプル数 |

**weighted avg と macro avg の乖離はクラス不均衡のシグナル：**

| 集計方法 | 重み | 特性 |
|---------|------|------|
| `macro avg` | 全クラス均等 | 少数クラスの性能が低いと大きく引き下げられる。各クラスを公平に評価したいときに使う |
| `weighted avg` | support（サンプル数）で重み付け | 多数クラスの性能を主に反映する。データの現実の分布を加味した評価 |

**実例（ネットワーク侵入検知・5クラス分類）：**

```
              precision    recall  f1-score   support
      Normal       0.99      1.00      1.00     15402
         DoS       1.00      1.00      1.00     10721
       Probe       0.99      1.00      1.00      2796
   Privilege       0.62      0.24      0.34        21   ← support 21 / f1 低い
      Access       0.96      0.92      0.94       764

    accuracy                           0.99     29704
   macro avg       0.91      0.83      0.85     29704   ← 少数クラスの影響が出る
weighted avg       0.99      0.99      0.99     29704   ← 多数クラスの高精度が支配
```

この結果の読み取り方：

- `weighted avg ≈ 0.99` / `macro avg ≈ 0.85` → **乖離が大きい = クラス不均衡**
- `Privilege` は support=21 しかなく、precision 0.62・recall 0.24 → モデルが Privilege を検出できていない
- overall accuracy 0.99 は「Normal/DoS を当てているだけ」の可能性が高い。単体で信頼しない

**support が小さいクラスへの対処：**

- support が 50 未満程度のクラスは指標が統計的に不安定（サンプルが 1〜2 個違うだけで大きく変わる）
- 対処法：追加サンプルの収集・SMOTE 等のオーバーサンプリング・そのクラスを除外した二値分類に切り替える
- セキュリティ用途では「稀だが危険なクラス（権限昇格等）」を見落とすリスクとして認識する

**回帰問題：**
- **MAE（平均絶対誤差）**：外れ値への影響が小さい。
- **MSE（平均二乗誤差）**：外れ値に敏感。勾配計算で使いやすい。
- **RMSE**：MSEの平方根。元データと同じ単位で解釈できる。
- **R²（決定係数）**：モデルがデータのばらつきをどれだけ説明できるか（1に近いほど良い）。

### 注意点・落とし穴

**過学習（Overfitting）と未学習（Underfitting）：**
- 過学習：訓練データには高精度だがテストデータに弱い → モデルが複雑すぎる
- 未学習：訓練データにもテストデータにも精度が出ない → モデルが単純すぎる
- **バイアス・バリアンストレードオフ**：モデルの複雑さを調整してバランスをとる

**対策：**
- 正則化（L1/L2）：過学習を抑制するためにパラメータに罰則を加える
- ドロップアウト：ニューラルネットワーク用の正則化手法
- 交差検証（Cross-Validation）：データを複数のfoldに分けて評価の信頼性を高める
- データ拡張（Data Augmentation）：学習データを水増しする

**データ漏洩（Data Leakage）：**
- テストデータの情報が訓練に混入すると、楽観的すぎる評価になる
- 前処理（標準化のscaler fit等）は必ず訓練データのみで行う
