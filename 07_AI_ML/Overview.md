# 機械学習 概要

## 機械学習の種類と選択基準

### 着火条件
与えられた問題がどの学習パラダイムに属するかを判断する必要があるとき。

### 観点・着眼点

**3つの主要パラダイム：**

- **教師あり学習（Supervised Learning）**：ラベル付きデータ（入力と正解のペア）がある。分類・回帰問題に対応。
- **教師なし学習（Unsupervised Learning）**：ラベルなしデータ。データの構造・パターン・クラスタを発見する。
- **強化学習（Reinforcement Learning）**：エージェントが環境と対話しながら報酬を最大化するよう学習する。

**問題の種類による分類：**

| 問題タイプ | 説明 | 例 |
|-----------|------|-----|
| 分類（Classification） | カテゴリを予測 | スパム判定、画像認識 |
| 回帰（Regression） | 連続値を予測 | 価格予測、気温予測 |
| クラスタリング | データをグループ化 | 顧客セグメント |
| 次元削減 | 特徴量を圧縮 | 可視化、ノイズ除去 |
| 異常検知 | 外れ値の検出 | 不正検知、障害検知 |
| 生成モデル | 新しいデータを生成 | 画像生成、テキスト生成 |

### 手順（問題解決の一般フロー）

1. **問題定義**：予測したいものは何か、ラベルがあるか確認
2. **データ収集・確認**：サイズ、特徴量の型、欠損値、クラスバランスを確認
3. **前処理**：正規化/標準化、欠損値処理、カテゴリ変数のエンコーディング
4. **モデル選択**：問題タイプ・データサイズ・解釈性の要件に基づいて選択
5. **訓練**：学習データでモデルをフィット
6. **評価**：テストデータで汎化性能を測定
7. **改善**：ハイパーパラメータチューニング、特徴量エンジニアリング

### 評価指標

**分類問題：**
- **Accuracy（正解率）** = (TP+TN) / 全サンプル数。クラスが不均衡な場合は不適。
- **Precision（適合率）** = TP / (TP+FP)。陽性予測の正確さ。偽陽性を減らしたいとき重視。
- **Recall（再現率）** = TP / (TP+FN)。実際の陽性を見逃さない能力。偽陰性を減らしたいとき重視。
- **F1スコア** = 2 × (Precision × Recall) / (Precision + Recall)。PrecisionとRecallのバランス指標。
- **混同行列（Confusion Matrix）**：TP/TN/FP/FNの全体像を把握。

**回帰問題：**
- **MAE（平均絶対誤差）**：外れ値への影響が小さい。
- **MSE（平均二乗誤差）**：外れ値に敏感。勾配計算で使いやすい。
- **RMSE**：MSEの平方根。元データと同じ単位で解釈できる。
- **R²（決定係数）**：モデルがデータのばらつきをどれだけ説明できるか（1に近いほど良い）。

### 注意点・落とし穴

**過学習（Overfitting）と未学習（Underfitting）：**
- 過学習：訓練データには高精度だがテストデータに弱い → モデルが複雑すぎる
- 未学習：訓練データにもテストデータにも精度が出ない → モデルが単純すぎる
- **バイアス・バリアンストレードオフ**：モデルの複雑さを調整してバランスをとる

**対策：**
- 正則化（L1/L2）：過学習を抑制するためにパラメータに罰則を加える
- ドロップアウト：ニューラルネットワーク用の正則化手法
- 交差検証（Cross-Validation）：データを複数のfoldに分けて評価の信頼性を高める
- データ拡張（Data Augmentation）：学習データを水増しする

**データ漏洩（Data Leakage）：**
- テストデータの情報が訓練に混入すると、楽観的すぎる評価になる
- 前処理（標準化のscaler fit等）は必ず訓練データのみで行う
