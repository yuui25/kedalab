# パーセプトロン・ニューラルネットワーク

## パーセプトロン（Perceptron）

### 着火条件
- 線形分離可能な2クラス分類問題
- ニューラルネットワークの基礎を理解する出発点

### 観点・着眼点

**構造：**
```
出力 = step(w₁x₁ + w₂x₂ + ... + wₙxₙ + b)
```
- 入力に重みをかけて足し合わせ、**ステップ関数**（閾値で0/1に変換）を適用
- 学習：誤分類があったときのみ重みを更新（パーセプトロン学習則）

**限界：**
- XOR問題を解けない（線形分離不可能）
- この限界を突破するために多層化（多層パーセプトロン）が必要

---

## ニューラルネットワーク（Neural Network）

### 着火条件
- 非線形な関係を学習する必要があるとき
- 画像・音声・テキストなど複雑な特徴を持つデータ
- 大量の学習データがある場合に特に効果的
- 従来の機械学習（SVM、決定木）で精度が出ない問題

### 観点・着眼点

**ニューロン（Neuron）の構造：**
```
z = w₁x₁ + w₂x₂ + ... + wₙxₙ + b  (線形結合)
a = f(z)                              (活性化関数を適用)
```
- パーセプトロンとの違い：ステップ関数以外の活性化関数（ReLU, sigmoid, tanh）を使う

**活性化関数の選択：**
| 関数 | 式 | 用途 | 特徴 |
|------|-----|------|------|
| ReLU | `max(0, z)` | 隠れ層（デフォルト） | 勾配消失に強い、計算が速い |
| Sigmoid | `1/(1+e⁻ᶻ)` | 2値分類の出力層 | 0〜1の確率出力 |
| Softmax | `eᶻⁱ/Σeᶻʲ` | 多クラス分類の出力層 | 全クラスの確率の和が1 |
| Tanh | `(eᶻ-e⁻ᶻ)/(eᶻ+e⁻ᶻ)` | 隠れ層（RNN等） | -1〜1の出力 |

**ネットワークの層構成：**
- **入力層**：特徴量を受け取る。ニューロン数 = 特徴量数
- **隠れ層**：中間的な表現を学習。層数・ニューロン数はハイパーパラメータ
- **出力層**：最終的な予測。分類はクラス数のニューロン+ソフトマックス

**学習（誤差逆伝播法 / Backpropagation）：**
1. 入力を順伝播して予測を計算
2. 損失（予測と正解の差）を計算
3. 損失の勾配を出力層から入力層に向けて逆伝播
4. 勾配降下法で重みを更新

**勾配降下法のバリアント：**
- **バッチGD**：全データで勾配を計算。安定だが遅い
- **確率的GD（SGD）**：1サンプルで勾配を計算。速いが不安定
- **ミニバッチGD**：数十〜数百サンプルで計算。最も一般的（バッチサイズ32〜256）
- **Adam**：適応的な学習率。ほとんどの問題でSGDより速く収束。現在のデフォルト選択肢

**正則化技術：**
- **ドロップアウト（Dropout）**：学習時にランダムにニューロンを無効化。過学習防止
- **バッチ正規化（Batch Normalization）**：各層の入力を正規化。学習を安定化・高速化
- **L2正則化（Weight Decay）**：重みの大きさにペナルティ

**オートエンコーダ：**
- エンコーダ：入力を低次元の潜在表現に圧縮
- デコーダ：潜在表現から元の入力を復元
- 用途：次元削減、ノイズ除去、異常検知（再構成誤差で検知）

### 手順（PyTorchの概要）

```python
import torch.nn as nn

class MLP(nn.Module):
    def __init__(self):
        super().__init__()
        self.layers = nn.Sequential(
            nn.Linear(input_dim, 128),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(128, 64),
            nn.ReLU(),
            nn.Linear(64, num_classes)
        )
    
    def forward(self, x):
        return self.layers(x)
```

### 注意点・落とし穴

- **勾配消失問題**：深いネットワークでは逆伝播時に勾配が小さくなり学習が停滞する。ReLUの使用・バッチ正規化・ResNetの残差接続で対応。
- **勾配爆発問題**：勾配が大きくなりすぎて不安定になる。勾配クリッピングで対応。
- **大量データが必要**：少ないデータでは過学習しやすい。転移学習（事前学習済みモデル）を活用。
- **ハイパーパラメータが多い**：学習率・バッチサイズ・層数・ニューロン数・ドロップアウト率など。体系的なチューニングが必要。
- **初期化が重要**：重みの初期値によって学習が大きく変わる。He初期化（ReLU用）やXavier初期化（sigmoid/tanh用）を使う。
