# 拡散モデル（Diffusion Models）

### 着火条件
- 高品質な画像・音声・動画の生成が必要なとき
- テキストから画像を生成したいとき（Text-to-Image）
- 画像の補完・編集・スタイル変換
- 生成画像の多様性と品質を両立させたいとき
- GAN（生成的敵対的ネットワーク）の代替を探しているとき

### 観点・着眼点

**動作原理（2フェーズ）：**

**1. 前向き過程（Forward Process / Diffusion）：**
```
x₀ → x₁ → x₂ → ... → xT（純粋なガウスノイズ）
```
- 元の画像 `x₀` に対して、T個のタイムステップで少しずつノイズを加えていく
- Tステップ後はほぼ完全なガウスノイズになる
- これは訓練データを作る固定のプロセス（学習するものではない）

**2. 逆向き過程（Reverse Process / Denoising）：**
```
xT（ノイズ）→ ... → x₁ → x₀（生成画像）
```
- ニューラルネットワーク（主にU-Net）が「このノイズはどこから来たか」を逆に予測
- 各ステップで少しずつノイズを除去していく
- **学習目標**：タイムステップtと noisy画像 `xₜ` を入力として、加えられたノイズを予測

**直感的な理解：**
- 彫刻家が大理石の塊から少しずつ削って像を作るイメージ（逆方向で考えれば）
- あるいは：砂嵐の映像から徐々に鮮明な画像を浮かび上がらせるイメージ

**Stable Diffusionの仕組み（Latent Diffusion）：**
- ピクセル空間ではなく**潜在空間（VAEの低次元表現）**で拡散を行う
- 計算コストを大幅削減（解像度の低い潜在空間で処理）
- テキスト条件付けにはCLIPなどのテキストエンコーダを使用
- CLIP：テキストと画像を同一の埋め込み空間に写像するモデル

**GAN（Generative Adversarial Network）との比較：**
| 特徴 | Diffusion Models | GAN |
|------|-----------------|-----|
| 学習安定性 | 安定（ノイズ除去の目標が明確） | 不安定（モード崩壊の問題） |
| 生成品質 | 高い | 高い（但し難しい） |
| 生成速度 | 遅い（多数のステップが必要） | 速い（1回のforward pass） |
| 多様性 | 高い | 低いことがある |
| テキスト条件付け | 容易 | 難しい |

**サンプリングの高速化：**
- **DDPM**：1000ステップ必要（遅い）
- **DDIM**：50〜100ステップに削減（確定的サンプリング）
- **LCM（Latent Consistency Model）**：4〜8ステップで生成

### 手順（Stable Diffusionの概念的な流れ）

```
テキストプロンプト
    ↓ CLIPテキストエンコーダ
テキスト埋め込みベクトル
    ↓ + ランダムノイズ（潜在空間）
[U-Net × Nステップのデノイズ]
    ← テキスト埋め込みで条件付け（Cross-Attention）
    ↓
クリーンな潜在表現
    ↓ VAEデコーダ
生成画像
```

**実用的な利用（Diffusers ライブラリ）：**
```python
from diffusers import StableDiffusionPipeline
import torch

pipe = StableDiffusionPipeline.from_pretrained(
    "runwayml/stable-diffusion-v1-5",
    torch_dtype=torch.float16
).to("cuda")

image = pipe(
    "a photo of a cat in cyberpunk style",
    num_inference_steps=50,
    guidance_scale=7.5
).images[0]
```

**Guidance Scale（`guidance_scale`）：**
- プロンプトとの一致度と多様性のトレードオフを制御
- 高い値（7〜15）：プロンプトに忠実だが多様性が低い
- 低い値（1〜5）：多様だがプロンプトから外れやすい

### 注意点・落とし穴

- **生成に時間がかかる**：多数のデノイズステップが必要。DDIM・LCM等の高速化手法を活用。
- **プロンプトエンジニアリングが重要**：ネガティブプロンプト（生成させたくない要素）の指定も有効。
- **著作権・倫理的問題**：実在する人物・著作物に似た画像生成が問題になる場合がある。
- **メモリ使用量が大きい**：フルサイズのStable Diffusionには少なくとも6〜8GBのVRAMが必要。
- **推論時の一貫性（シード固定）**：同じシードを使えば同じ画像を再現できる。
