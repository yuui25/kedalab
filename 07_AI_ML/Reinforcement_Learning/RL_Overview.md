# 強化学習（Reinforcement Learning）概要

### 着火条件
- エージェントが環境と対話しながら意思決定を最適化したい問題
- ゲームAI、ロボット制御、自動売買、推薦システムの最適化
- 正解ラベルはないが「報酬」として定義できる目標がある
- 逐次的な行動が必要で、現在の選択が将来の状態に影響する

### 観点・着眼点

**強化学習の構成要素：**

| 要素 | 説明 |
|------|------|
| **エージェント（Agent）** | 学習・意思決定する主体 |
| **環境（Environment）** | エージェントが相互作用する対象 |
| **状態（State, s）** | 現在の環境の状況 |
| **行動（Action, a）** | エージェントが選択できる操作 |
| **報酬（Reward, r）** | 行動の結果として得られるスカラー値（正/負） |
| **方策（Policy, π）** | 状態に対して行動を決定するルール `π(a|s)` |
| **価値関数（Value Function）** | ある状態（または状態・行動ペア）から得られる将来の累積報酬の期待値 |

**割引累積報酬（Return）：**
```
G_t = r_{t+1} + γ·r_{t+2} + γ²·r_{t+3} + ...
```
- `γ`（割引率）：0〜1。将来の報酬を現在どれだけ重視するか。γ=0なら即時報酬のみ、γ→1なら遠い将来も考慮。

**価値関数の種類：**
- **状態価値関数 V(s)**：状態sにいるときの期待累積報酬
- **行動価値関数 Q(s,a)**：状態sで行動aをとったときの期待累積報酬

**強化学習の分類：**
- **モデルフリー vs モデルベース**：環境のモデルを学習するかどうか
- **Value-Based（価値ベース）**：Q値を学習して、最大Q値の行動を選ぶ（Q-learning、DQN）
- **Policy-Based（方策ベース）**：方策を直接最適化（REINFORCE、PPO）
- **Actor-Critic**：価値関数と方策の両方を学習

**探索と活用のトレードオフ（Exploration vs Exploitation）：**
- 活用：現在知っている最良の行動を選ぶ
- 探索：新しい行動を試して、より良い行動を見つける
- **ε-greedy**：確率εでランダムに探索、1-εで最善行動を選択

### 注意点・落とし穴

- **報酬設計（Reward Shaping）が難しい**：不適切な報酬設計だと意図しない行動を学習する（例：ゲームのスコアを最大化するために不正なパターンを発見）
- **サンプル効率の悪さ**：多くの試行錯誤が必要。現実世界でのロボット学習では時間・コストがかかる
- **局所解への収束**：確率的な最適化のため、大域的最適解に到達する保証がない
- **環境のシミュレーション精度**：シミュレータで学習した方策が実環境で機能しないことがある（sim-to-gapの問題）
