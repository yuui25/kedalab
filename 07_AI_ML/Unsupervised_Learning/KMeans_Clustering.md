# K-Meansクラスタリング

### 着火条件
- ラベルなしデータをグループに分けたいとき
- 顧客セグメンテーション、文書クラスタリング、画像圧縮
- クラスタ数Kの見当がある（または試行できる）とき
- 球状のクラスタが期待されるとき

### 観点・着眼点

**動作原理（反復アルゴリズム）：**
1. K個のセントロイド（重心）をランダムに初期化
2. 各データ点を最も近いセントロイドに割り当て
3. 各クラスタのデータ点の平均でセントロイドを更新
4. セントロイドが変化しなくなるまで2〜3を繰り返す

**最適なK値の決定：**
- **エルボー法（Elbow Method）**：KごとにWCSS（クラスタ内二乗和）をプロット。減少率が鈍化する「肘」のポイントが最適K
- **シルエットスコア**：クラスタ内の凝集度とクラスタ間の分離度を測定。1に近いほど良い

**初期化の改善：**
- `k-means++`：最初のセントロイドをデータ分布を考慮して選ぶ。収束が速く安定。scikit-learnのデフォルト

### 手順

1. データの標準化（K-Meansはユークリッド距離を使うのでスケールの影響を受ける）
2. エルボー法でKを決定
   ```python
   wcss = []
   for k in range(1, 11):
       km = KMeans(n_clusters=k, init='k-means++', n_init=10)
       km.fit(X)
       wcss.append(km.inertia_)
   ```
3. `KMeans(n_clusters=k).fit(X)`でクラスタリング
4. `labels_`でクラスタの割り当てを取得
5. `cluster_centers_`でセントロイドの位置を確認
6. シルエットスコアで品質を評価

### 注意点・落とし穴

- **K値の事前指定が必要**：適切なKが不明な場合は複数試して評価する。
- **初期値依存性**：ランダム初期化だと局所解に陥ることがある。`n_init=10`で複数回実行して最良を選択（k-means++でほぼ解消）。
- **球状クラスタの仮定**：細長いクラスタや密度の異なるクラスタには弱い。GMM（混合ガウスモデル）やDBSCANを検討。
- **外れ値への感度**：外れ値がセントロイドを引き寄せる。前処理で外れ値を除去するか、K-Mediansを使用。
- **カテゴリ変数への不適**：ユークリッド距離が意味をなさない。K-Modesアルゴリズムを検討。
