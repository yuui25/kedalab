# 主成分分析（PCA: Principal Component Analysis）

### 着火条件
- 高次元データの次元削減が必要なとき
- 特徴量間に相関があり、冗長な情報を圧縮したいとき
- データの可視化（2D/3Dへの投影）
- SVMやニューラルネットワークの前処理として次元を減らしたいとき
- ノイズ除去をしたいとき

### 観点・着眼点

**動作原理：**
- データの分散が最大になる方向（**主成分**）を見つけて、その方向に投影する
- 第1主成分：分散が最大の方向
- 第2主成分：第1主成分と直交し、残りの分散が最大の方向
- 以降、互いに直交する主成分を順番に求める

**固有値分解：**
- 共分散行列の固有値・固有ベクトルを計算
- 固有値が大きい固有ベクトル = 分散が大きい主成分（重要な方向）

**累積寄与率（Explained Variance Ratio）：**
- 各主成分が全分散の何%を説明するかを示す
- 累積で80〜95%を超えるK個の主成分を残すのが一般的な目安

### 手順

1. データの標準化（必須：スケールが異なると分散の大きな特徴量が支配的になる）
2. PCAで主成分を計算
   ```python
   from sklearn.decomposition import PCA
   pca = PCA(n_components=0.95)  # 累積寄与率95%を保持する成分数を自動選択
   X_reduced = pca.fit_transform(X_scaled)
   ```
3. `explained_variance_ratio_`で各成分の寄与率を確認
4. スクリープロット（寄与率の減少カーブ）で成分数を視覚的に確認
5. 2成分に削減して散布図で可視化

### 注意点・落とし穴

- **線形変換のみ**：非線形な構造を持つデータには弱い。t-SNEやUMAPを代わりに使う（ただし可視化専用）。
- **解釈性の喪失**：主成分は元の特徴量の線形結合なので、「第1主成分が何を意味するか」が直接わからなくなる。
- **スケールへの依存**：必ず標準化してから適用する。
- **情報の損失**：次元を削減することで一部の情報は失われる。削減後の精度低下を必ず確認する。
- **外れ値への感度**：外れ値が分散を大きく歪め、主成分の方向が変わることがある。
