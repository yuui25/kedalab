# 線形回帰 / ロジスティック回帰

## 線形回帰（Linear Regression）

### 着火条件
- 連続値を予測したい（回帰問題）
- 入力特徴量と出力の関係が線形（または線形に近い）と仮定できる
- 解釈性が重要で「どの特徴量がどれだけ影響するか」を知りたい

### 観点・着眼点

**モデルの形式：**
```
ŷ = w₀ + w₁x₁ + w₂x₂ + ... + wₙxₙ
```
- `wᵢ`：重み（係数）。各特徴量の影響の大きさ
- `w₀`：バイアス（切片）
- 目的：実際の値 `y` と予測値 `ŷ` の差（残差）を最小化

**損失関数：**
- MSE（平均二乗誤差）を最小化する
- 解析解：最小二乗法（Ordinary Least Squares）で重みを直接計算できる
- 大規模データでは勾配降下法（Gradient Descent）を使う

**正則化バリアント：**
- **Ridge（L2正則化）**：全重みを小さくする。多重共線性に強い
- **Lasso（L1正則化）**：不要な特徴量の重みを0にする。特徴量選択の効果
- **ElasticNet**：L1とL2の組み合わせ

### 手順

1. データの正規化/標準化（スケールを揃える）
2. 学習データでモデルをfitする
3. 係数（`coef_`）でどの特徴量が重要かを確認
4. テストデータでMAE/RMSE/R²を評価
5. 残差プロットで仮定（線形性・等分散性）を確認

### 注意点・落とし穴

- **線形性の仮定**：入力と出力の関係が非線形な場合は精度が出ない。多項式特徴量を追加して対応。
- **多重共線性**：特徴量同士が強く相関していると係数が不安定。Ridgeで対応。
- **外れ値への感度**：MSEは二乗誤差のため外れ値の影響が大きい。MAEを使うか外れ値処理を先に行う。
- **スケールの影響**：特徴量のスケールが異なると収束が遅い。StandardScalerで標準化する。

## ロジスティック回帰（Logistic Regression）

### 着火条件
- カテゴリ（クラス）を予測したい（分類問題）
- 2クラス分類（バイナリ）または多クラス分類
- 確率として出力が必要な場合（「陽性の確率は何%か」を知りたい）
- ベースラインモデルとして最初に試す

### 観点・着眼点

**モデルの形式：**
```
P(y=1|x) = σ(w₀ + w₁x₁ + ... + wₙxₙ)
```
- **シグモイド関数** `σ(z) = 1/(1+e⁻ᶻ)` で確率（0〜1）に変換
- 決定境界：P > 0.5 なら陽性、P ≤ 0.5 なら陰性（閾値は調整可能）

**損失関数：**
- 交差エントロピー損失（Cross-Entropy Loss / Log Loss）を最小化
- 線形回帰と同様にMSEを使わないのは、分類ではMSEが非凸になるため

**多クラス拡張：**
- **One-vs-Rest（OvR）**：各クラスを「このクラスかそれ以外か」で二値分類
- **ソフトマックス（Multinomial）**：全クラスの確率を同時に計算。合計が1になる

### 手順

1. データの標準化
2. `LogisticRegression()`でfit
3. `predict_proba()`で確率を確認（0.5以外の閾値にしたい場合に使用）
4. 混同行列・Precision/Recall/F1で評価
5. クラスが不均衡な場合は`class_weight='balanced'`を指定

### 注意点・落とし穴

- **線形分離可能性の仮定**：決定境界が線形でない問題には弱い。カーネルSVMや決定木を検討。
- **クラス不均衡**：多数クラスに偏った予測になる。`class_weight='balanced'`や過小サンプリングで対応。
- **確率の信頼性**：ロジスティック回帰の確率はキャリブレーションが比較的良いが、必ずしも正確ではない。
- **特徴量のスケール**：正則化の影響を均等にするため標準化が必須。
