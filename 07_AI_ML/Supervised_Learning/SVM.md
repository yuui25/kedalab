# サポートベクターマシン（SVM）

### 着火条件
- 高次元データの分類（特徴量の数がサンプル数より多い、またはそれに近い）
- テキスト分類・画像分類などで特徴量空間が広いとき
- 2クラス分類でマージンの最大化が重要なとき
- 線形分離不可能なデータにカーネルを使って対応したいとき

### 観点・着眼点

**動作原理：**
- 2クラスを分けるマージン（余白）を最大化する決定境界を見つける
- **サポートベクター**：決定境界に最も近いデータ点。この点だけがモデルに影響する
- **マージン** = 決定境界から各クラスの最近傍点までの距離の合計

**ハードマージンとソフトマージン：**
- **ハードマージン**：誤分類を一切許さない。完全線形分離可能なデータのみ適用可能
- **ソフトマージン**：誤分類を少し許容してマージンを広げる。`C`パラメータで制御
  - C が大きい：誤分類を厳しく罰する（過学習リスク）
  - C が小さい：誤分類を許容してマージンを広くする（未学習リスク）

**カーネルトリック（非線形分類）：**
- 低次元で線形分離不可能なデータを高次元空間に写像して線形分離する
- **RBFカーネル（ガウシアン）**：最も広く使われる。局所的なパターンを捉える
- **多項式カーネル**：特徴量の多項式組み合わせ
- `gamma`パラメータ：RBFの影響範囲。大きいと過学習しやすい

### 手順

1. 特徴量を標準化（SVMはスケールに敏感）
2. カーネルの選択：線形分離可能なら`kernel='linear'`、それ以外は`kernel='rbf'`を試す
3. グリッドサーチで`C`と`gamma`をチューニング
   ```python
   from sklearn.model_selection import GridSearchCV
   param_grid = {'C': [0.1, 1, 10], 'gamma': ['scale', 'auto', 0.1]}
   ```
4. テストデータで評価

**回帰（SVR）：**
- 予測値が ε（イプシロン）チューブの内側に収まるように学習
- `epsilon`パラメータ：許容誤差の幅

### 注意点・落とし穴

- **スケールへの感度**：正規化・標準化を必ず行う。特徴量のスケールがそのままマージン計算に影響する。
- **大規模データでの遅さ**：学習の時間計算量が O(n²)〜O(n³)。サンプル数が数万を超えると遅くなる。LinearSVCを検討。
- **確率出力の非対応**：基本的に確率を出力しない。`probability=True`で有効化できるが計算コストが増える。
- **カーネル選択の難しさ**：RBFは万能だが、解釈性がなくなる。問題に合わせてチューニングが必要。
- **多クラス分類**：One-vs-Rest（OvR）またはOne-vs-One（OvO）で対応。OvOがScikit-learnのデフォルト。
